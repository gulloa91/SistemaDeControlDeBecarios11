Los permisos que se han definido para el sistema son los siguientes:

1)   Módulo para Administración de Becarios:

a)      Becario Completo: Tiene permiso para consultar los becarios en el sistema, insertar nuevos becarios, modificarlos y eliminarlos. Muestra una lista al usuario con todos los becarios. El usuario puede selecionar un becario para consultarlo, modificarlo o eliminarlo. También presenta la opción de insertar nuevos becarios en el sistema.

b)      Becario Parcial: Muestra solo la información personal del usuario. Permite al usuario hacer modificaciones sobre algunos de sus datos personales.

c)       Sin acceso a Becario: no puede realizar ninguna operación que involucre los datos personales de algún becario.

2)   Módulo para Administración de Encargados

a)      Encargado Completo: Tiene permiso para consultar los encargados en el sistema, insertar nuevos encargados, modificarlos y eliminarlos. Muestra una lista al usuario con todos los encargados. El usuario puede selecionar un becario para consultarlo, modificarlo o eliminarlo. También presenta la opción de insertar nuevos encargados en el sistema.

b)      Encargado Parcial: Muestra solo la información personal del usuario. Permite al usuario hacer modificaciones sobre algunos de sus datos personales.

c)       Sin acceso a Encargado: no puede realizar ninguna operación que involucre los datos personales de algún encargado.

3)   Módulo de Control de Horas

a)      Control Horas Estudiante: Permite al becario reportar al encargado las horas trabajadas. Permite consultar reportes de horas anteriores. El usuario reporta la cantidad de horas trabajadas, la fecha en que las reporta, y puede enviar un comentario justificando su trabajo.

b)      Control Horas Encargado: Permite al encargado aprobar o rechazar las horas reportadas por los becarios que tiene asignados. Si rechaza las horas se habilita la posibilidad de u comentario para justificar la acción

c)       Sin Acceso Control de Horas: no puede realizar ninguna operación que involucre el proceso de interacción de control de horas.

4)   Módulo de Asignaciones

a)      Asignación de horas Completa: Tiene permiso para insertar, modificar, eliminar y consultar todas las asignaciones en el sistema .Muestra una lista al usuario con todas las asignaciones. El usuario puede selecionar una asignación para consultarla, modificarla o eliminarla. También presenta la opción de insertar nuevas asignaciones en el sistema.

b)      Asignación de horas Encargado: El Encargado puede confirmar/cancelar las asignaciones de los becarios que se le han asignado. Esto se hace solo una vez, luego de que la asignación es creada.

c)       Asignación de horas Becario: El Becario puede confirmar/cancelar la asignación con el encargado que se le ha asignado.Puede consultar tanto la identidad de su encargado en el actual semestre, como el estado de la asignación. Esto se hace solo una vez, luego de que la asignación es creada.

5)   Módulo para Administración de Perfiles

a)      Vista Completa de Perfiles: Tiene permiso para consultar los perfiles en el sistema, insertar nuevos perfiles, modificarlos y eliminarlos. Al insertar o modificar un perfil escoge los permisos que van a estar asociados al perfil.

6)   Módulo para Administración de Cuentas

a)      Cuentas Completo: Tiene permiso para consultar todas las cuentas en el sistema, insertar nuevas cuentas y eliminarlas. Puede modificar únicamente las constraseñas.

b)      Cuentas Parcial: Muestra únicamente la información personal de la cuenta del usuario. Puede modificar la contraseña.

c)       Sin acceso a Cuenta: no puede realizar ninguna operación que involucre alguna cuenta en el sistema.

7)   Módulo para Administración de Reportes

a)      Reportes: Muestra el módulo de Reportes. Permite reportar historiales, asignaciones, becarios y encargados. Concede acceso a los reportes de los que el sistema tiene capacidad para generar, independientemente del resto de permisos asignados. Algunos reportes permiten generar un pdf de la cosnulta.
